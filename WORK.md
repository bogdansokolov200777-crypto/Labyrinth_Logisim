Готовые модули

1. BTN_INPUT (Ввод)

Функция: Превращает нажатие кнопки в единичный импульс (1 такт).
Логика: Использован детектор фронта на D-триггере (Q + NOT(Q_prev)), чтобы игрок не "летел" через всю карту при удержании кнопки.

2. PLAYER_POS (Позиция)
Функция: Хранит текущие координаты игрока (POS_X, POS_Y).
Логика:
Использует регистры (Registers) для хранения.
Splitter собирает 12-битный адрес {Y, X} для обращения к памяти.
Мультиплексоры управляют возвратом игрока на START при сбросе.

4. MAP_MEM (Карта)
Функция: Хранит лабиринт.
Логика:
Выдает тип клетки (CELL_TYPE) для проверки логики.
Позволяет стирать объекты (ключи, бонусы) по сигналу WE.
Адрес ADDR идет от игрока (текущая клетка) или от Move Logic (следующая клетка).

4. MOVE_LOGIC 
1.Фильтр кнопок через MOVE_ENABLE и GAME_STATE.
2.Сумматоры/Вычитатели для новых координат. Компораторы для проверки границ карты (чтобы не уйти за 0 или 63).
3.Сплиттеры формируют адрес ADDR_NEXT, декодер распознает тип клетки (Стена, Выход, Ключ и т.д.).
4.Логические вентили решают, можно ли сделать шаг (DO_MOVE), формируют новые координаты (NEW_X/Y) и генерируют события (EV_...).


Адресация: 12-битный адрес строится простой конкатенацией координат: Address = {Y[5:0], X[5:0]}.
Безопасность: Добавлена проверка границ (BOUND_R/L/U/D), предотвращающая переполнение регистров на краях карты.
Гибкость: Выход MOVE_LOGIC закрытый (IS_EXIT_C) открывается через внешний флаг EXIT_OPEN.
Статус сборки
Main: Схема main собрана и объединяет все блоки.
Тестовый режим: Входы от других участников (MOVE_ENABLE, EXIT_OPEN) пока заглушены константами (1, 0).
