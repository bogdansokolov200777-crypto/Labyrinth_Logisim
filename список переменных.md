# Переменные проекта «Лабиринт» для Logisim

## Регистры (хранение значений)

| Переменная | Разрядность | Назначение |
|------------|-------------|------------|
| `POS_X` | 6 бит | координата X игрока (0-63) |
| `POS_Y` | 6 бит | координата Y игрока (0-63) |
| `START_X` | 6 бит | координата X старта |
| `START_Y` | 6 бит | координата Y старта |
| `EXIT_X` | 6 бит | координата X выхода |
| `EXIT_Y` | 6 бит | координата Y выхода |
| `KEY_COLLECTED` | 4 бита | количество собранных ключей (до 15) |
| `KEY_COUNT_TOTAL` | 4 бита | общее количество ключей на карте |
| `FREEZE_COUNTER` | 3 бита | счётчик оставшихся ходов заморозки (0-7) |
| `BONUS_COUNTER` | 5 битов | счётчик действия бонуса (в тактах/ходах) |
| `TIMER_SEC` | 6 битов | секунды текущего времени (0-59) |
| `TIMER_MIN` | 6 битов | минуты текущего времени (0-59) |
| `BEST_TIME_SEC` | 6 битов | рекордные секунды |
| `BEST_TIME_MIN` | 6 битов | рекордные минуты |
| `MOB1_X` | 6 бит | координата X моба 1 |
| `MOB1_Y` | 6 бит | координата Y моба 1 |
| `MOB2_X` | 6 бит | координата X моба 2 (опционально) |
| `MOB2_Y` | 6 бит | координата Y моба 2 (опционально) |
| `MOB_DIR` | 2 бита | направление движения моба (0-3) |

---

## ОЗУ (RAM) — массивы данных

| Массив | Размер | Разрядность | Назначение |
|--------|-------|-------------|------------|
| `MAP_DATA` | 4096 ячеек (64×64) | 4 бита | тип каждой клетки: 0=проход, 1=стена, 2=ключ, 3=выход(закр), 4=выход(откр), 5=ловушка, 6=бонус, 7=старт |
| `MAP_BACKUP` | 4096 ячеек | 4 бита | резервная копия карты для сброса (RESET) |

---

## Счётчики (Counter)

| Переменная | Разрядность | Назначение |
|------------|-------------|------------|
| `TIMER_COUNTER` | 30 бит | счётчик тактов для формирования секунд (при 1 Гц — делитель) |
| `MOB_MOVE_COUNTER` | 24 бита | счётчик для задания скорости движения мобов |
| `BLINK_COUNTER` | 24 бита | счётчик для мигания клеток (ключ, ловушка, старт) |
| `FREEZE_TIMER` | 3 бита | аппаратный счётчик ходов заморозки (декремент при каждом такте) |

---

## Флаги (1-битные регистры)

| Переменная | Назначение |
|------------|------------|
| `GAME_STATE` | 0=играем, 1=победа, 2=поражение (2 бита) |
| `LIFE_FLAG` | 0=жив, 1=мёртв |
| `MOVE_ENABLE` | 0=заморожен, 1=может двигаться |
| `EXIT_OPEN` | 0=выход закрыт, 1=открыт |
| `BONUS_ACTIVE` | 0=бонус не активен, 1=активен |
| `TIMER_RUN` | 0=таймер остановлен, 1=идёт |
| `TIME_LIMIT_MODE` | 0=лимит выключен, 1=включён |
| `FOG_ENABLE` | 0=туман выключен, 1=включён |
| `BTN_UP` | кнопка вверх (с антидребезгом) |
| `BTN_DOWN` | кнопка вниз |
| `BTN_LEFT` | кнопка влево |
| `BTN_RIGHT` | кнопка вправо |
| `BTN_RESET` | полный сброс |
| `BTN_RESTART` | перезапуск партии |
| `WIN_FLAG` | флаг победы для отображения экрана |
| `LOSE_FLAG` | флаг поражения для отображения экрана |

---

## Демультиплексоры / Комбинационная логика

| Компонент | Назначение |
|-----------|------------|
| `CELL_TYPE_DECODER` | декодирует тип клетки из MAP_DATA в отображение на дисплее |
| `MOVE_LOGIC` | проверяет возможность движения на целевую клетку (стена, закрытый выход, граница) |
| `COLLISION_DETECTOR` | детектор столкновения игрока с мобом |
| `KEY_COMPARATOR` | сравнивает KEY_COLLECTED и KEY_COUNT_TOTAL для открытия выхода |
| `TIMER_COMPARATOR` | сравнивает текущее время с рекордом при победе |
| `RADIUS_CONTROL` | управляет радиусом видимости (базовый / с бонусом) |
| `VGA_CONTROLLER` | формирует изображение на экране (64×64 клетки) |

---

## Специальные блоки Logisim

| Блок | Назначение |
|------|------------|
| **RAM (4096×4)** | MAP_DATA |
| **RAM (4096×4)** | MAP_BACKUP |
| **Register (6-bit)** | POS_X, POS_Y и другие координаты |
| **Counter (30-bit)** | TIMER_COUNTER (делитель частоты) |
| **Counter (3-bit)** | FREEZE_COUNTER (авто-декремент) |
| **ROM (карта прошивки)** | начальная карта лабиринта (загружается в RAM при RESET) |
| **7-segment display** | отображение KEY_COLLECTED / KEY_COUNT_TOTAL |
| **7-segment display** | отображение TIMER_MIN и TIMER_SEC |
| **7-segment display** | отображение BEST_TIME_MIN и BEST_TIME_SEC |
