# Список действий после событий (полная версия)

## Базовые события (управление и движение)

### При EV_RESET (полный сброс/начало игры)
- Установить `POS_X = START_X`
- Установить `POS_Y = START_Y`
- Установить `GAME_STATE = 0` (игра активна)
- Установить `LIFE_FLAG = 0` (игрок жив)
- Установить `MOVE_ENABLE = 1` (управление разрешено)
- Установить `FREEZE_COUNTER = 0` (нет заморозки)
- Установить `KEY_COLLECTED = 0`
- Установить `EXIT_OPEN = 0` (выход закрыт)
- Установить `BONUS_ACTIVE = 0`
- Установить `BONUS_COUNTER = 0`
- Установить `TIMER_RUN = 1` (таймер запущен)
- Установить `TIMER_SEC = 0`
- Установить `TIMER_MIN = 0`
- Восстановить `MAP_DATA` из `MAP_BACKUP` (исходная карта)
- Обновить дисплей (показать карту, позицию игрока, счётчики)

### При EV_RESTART (перезапуск партии)
- Установить `POS_X = START_X`
- Установить `POS_Y = START_Y`
- Установить `LIFE_FLAG = 0`
- Установить `MOVE_ENABLE = 1`
- Установить `FREEZE_COUNTER = 0`
- Установить `KEY_COLLECTED = 0`
- Установить `EXIT_OPEN = 0`
- Установить `BONUS_ACTIVE = 0`
- Установить `BONUS_COUNTER = 0`
- Восстановить `MAP_DATA` из `MAP_BACKUP`
- `TIMER_RUN` не изменять (таймер продолжает идти)
- `TIMER_SEC` и `TIMER_MIN` не сбрасывать
- Обновить дисплей

### При EV_BUTTON_UP
- Если `GAME_STATE != 0` → выход (игра не активна)
- Если `MOVE_ENABLE == 0` → выход (игрок заморожен)
- Вычислить новую позицию: `new_y = POS_Y - 1`
- Если `new_y < 0` → вызвать `EV_BOUNDARY_HIT`, выход
- Проверить тип клетки по координатам (`POS_X`, `new_y`) в `MAP_DATA`
- Если клетка = стена (1) → вызвать `EV_WALL_HIT`
- Если клетка = выход закрытый (3) и `EXIT_OPEN == 0` → вызвать `EV_EXIT_CLOSED_HIT`
- Если клетка = выход открытый (4) или (клетка = выход закрытый и `EXIT_OPEN == 1`) → вызвать `EV_EXIT_REACHED`
- Если клетка = ключ (2) → `POS_Y = new_y`, вызвать `EV_KEY_PICKUP`
- Если клетка = ловушка (5) → `POS_Y = new_y`, вызвать `EV_TRAP_TRIGGER`
- Если клетка = бонус (6) → `POS_Y = new_y`, вызвать `EV_BONUS_PICKUP`
- Если клетка = проход (0) → `POS_Y = new_y`, вызвать `EV_PLAYER_MOVE`

### При EV_BUTTON_DOWN
- Если `GAME_STATE != 0` → выход
- Если `MOVE_ENABLE == 0` → выход
- Вычислить новую позицию: `new_y = POS_Y + 1`
- Если `new_y > 63` → вызвать `EV_BOUNDARY_HIT`, выход
- Проверить тип клетки по координатам (`POS_X`, `new_y`) в `MAP_DATA`
- Если клетка = стена (1) → вызвать `EV_WALL_HIT`
- Если клетка = выход закрытый (3) и `EXIT_OPEN == 0` → вызвать `EV_EXIT_CLOSED_HIT`
- Если клетка = выход открытый (4) или (клетка = выход закрытый и `EXIT_OPEN == 1`) → вызвать `EV_EXIT_REACHED`
- Если клетка = ключ (2) → `POS_Y = new_y`, вызвать `EV_KEY_PICKUP`
- Если клетка = ловушка (5) → `POS_Y = new_y`, вызвать `EV_TRAP_TRIGGER`
- Если клетка = бонус (6) → `POS_Y = new_y`, вызвать `EV_BONUS_PICKUP`
- Если клетка = проход (0) → `POS_Y = new_y`, вызвать `EV_PLAYER_MOVE`

### При EV_BUTTON_LEFT
- Если `GAME_STATE != 0` → выход
- Если `MOVE_ENABLE == 0` → выход
- Вычислить новую позицию: `new_x = POS_X - 1`
- Если `new_x < 0` → вызвать `EV_BOUNDARY_HIT`, выход
- Проверить тип клетки по координатам (`new_x`, `POS_Y`) в `MAP_DATA`
- Если клетка = стена (1) → вызвать `EV_WALL_HIT`
- Если клетка = выход закрытый (3) и `EXIT_OPEN == 0` → вызвать `EV_EXIT_CLOSED_HIT`
- Если клетка = выход открытый (4) или (клетка = выход закрытый и `EXIT_OPEN == 1`) → вызвать `EV_EXIT_REACHED`
- Если клетка = ключ (2) → `POS_X = new_x`, вызвать `EV_KEY_PICKUP`
- Если клетка = ловушка (5) → `POS_X = new_x`, вызвать `EV_TRAP_TRIGGER`
- Если клетка = бонус (6) → `POS_X = new_x`, вызвать `EV_BONUS_PICKUP`
- Если клетка = проход (0) → `POS_X = new_x`, вызвать `EV_PLAYER_MOVE`

### При EV_BUTTON_RIGHT
- Если `GAME_STATE != 0` → выход
- Если `MOVE_ENABLE == 0` → выход
- Вычислить новую позицию: `new_x = POS_X + 1`
- Если `new_x > 63` → вызвать `EV_BOUNDARY_HIT`, выход
- Проверить тип клетки по координатам (`new_x`, `POS_Y`) в `MAP_DATA`
- Если клетка = стена (1) → вызвать `EV_WALL_HIT`
- Если клетка = выход закрытый (3) и `EXIT_OPEN == 0` → вызвать `EV_EXIT_CLOSED_HIT`
- Если клетка = выход открытый (4) или (клетка = выход закрытый и `EXIT_OPEN == 1`) → вызвать `EV_EXIT_REACHED`
- Если клетка = ключ (2) → `POS_X = new_x`, вызвать `EV_KEY_PICKUP`
- Если клетка = ловушка (5) → `POS_X = new_x`, вызвать `EV_TRAP_TRIGGER`
- Если клетка = бонус (6) → `POS_X = new_x`, вызвать `EV_BONUS_PICKUP`
- Если клетка = проход (0) → `POS_X = new_x`, вызвать `EV_PLAYER_MOVE`

### При EV_PLAYER_MOVE (успешное перемещение)
- Обновить дисплей (новые координаты, карта)
- Вызвать `EV_FOG_UPDATE` (обновление зоны видимости)
- Вызвать `EV_DISPLAY_UPDATE`
- Проверить, совпадают ли `POS_X` и `POS_Y` с `EXIT_X` и `EXIT_Y`
- Если совпадают и `EXIT_OPEN == 1` → вызвать `EV_EXIT_REACHED`
- Если совпадают и `EXIT_OPEN == 0` → ничего не делать (игрок на месте выхода, но он закрыт)

### При EV_WALL_HIT (удар о стену)
- Игрок остаётся на месте (`POS_X`, `POS_Y` не меняются)
- Зажечь индикатор ошибки на короткое время

### При EV_BOUNDARY_HIT (попытка выйти за границу)
- Игрок остаётся на месте
- Зажечь индикатор ошибки на короткое время

## События взаимодействия с клетками

### При EV_KEY_PICKUP (подобран ключ)
- Установить тип клетки в `MAP_DATA` по координатам игрока = 0 (проход)
- `KEY_COLLECTED = KEY_COLLECTED + 1`
- Обновить дисплей (счётчик ключей)
- Если `KEY_COLLECTED == KEY_COUNT_TOTAL` → вызвать `EV_ALL_KEYS_COLLECTED`
- Вызвать `EV_PLAYER_MOVE`

### При EV_ALL_KEYS_COLLECTED (собраны все ключи)
- Установить `EXIT_OPEN = 1`
- Если клетка выхода в `MAP_DATA` имеет тип 3 (закрытый выход) → изменить на тип 4 (открытый выход)
- Зажечь индикатор открытия выхода
- Обновить дисплей

### При EV_TRAP_TRIGGER (ловушка активирована)
- Вызвать `EV_FREEZE_START`

### При EV_FREEZE_START (начало заморозки)
- Установить `MOVE_ENABLE = 0`
- Установить `FREEZE_COUNTER = 3` (на 3 хода)
- Зажечь индикатор заморозки

### При EV_FREEZE_END (окончание заморозки)
- Установить `MOVE_ENABLE = 1`
- Установить `FREEZE_COUNTER = 0`
- Погасить индикатор заморозки
- Обновить дисплей

### При EV_BONUS_PICKUP (подобран бонус)
- Установить тип клетки в `MAP_DATA` по координатам игрока = 0 (проход)
- Установить `BONUS_ACTIVE = 1`
- Установить `BONUS_COUNTER = 100` (например, 100 ходов или тактов)
- Вызвать `EV_RADIUS_EXPAND`
- Обновить дисплей
- Вызвать `EV_PLAYER_MOVE`

### При EV_BONUS_EXPIRED (бонус истёк)
- Установить `BONUS_ACTIVE = 0`
- Установить `BONUS_COUNTER = 0`
- Вызвать `EV_RADIUS_NORMALIZE`
- Обновить дисплей

### При EV_EXIT_REACHED (достигнут открытый выход)
- Установить `GAME_STATE = 1` (победа)
- Установить `TIMER_RUN = 0` (остановить таймер)
- Вызвать `EV_GAME_WIN`

### При EV_EXIT_CLOSED_HIT (попытка войти в закрытый выход)
- Игрок остаётся на месте
- Зажечь индикатор ошибки (выход закрыт)

## События мобов и столкновений

### При EV_MOB_MOVE (моб совершил шаг)
- Вычислить новую позицию моба согласно его логике движения
- Проверить, совпадает ли новая позиция с `POS_X`, `POS_Y`
- Если совпадает → вызвать `EV_MOB_COLLISION`
- Обновить дисплей

### При EV_MOB_COLLISION (столкновение с мобом)
- Если `LIFE_FLAG == 0`:
  - Установить `LIFE_FLAG = 1`
  - Вызвать `EV_PLAYER_DEATH`

### При EV_PLAYER_DEATH (игрок убит)
- Установить `MOVE_ENABLE = 0` (временно заблокировать движение)
- Вызвать `EV_PLAYER_RESPAWN`

### При EV_PLAYER_RESPAWN (возрождение игрока)
- Установить `POS_X = START_X`
- Установить `POS_Y = START_Y`
- Установить `LIFE_FLAG = 0`
- Установить `KEY_COLLECTED = 0`
- Установить `EXIT_OPEN = 0`
- Установить `FREEZE_COUNTER = 0`
- Установить `BONUS_ACTIVE = 0`
- Установить `BONUS_COUNTER = 0`
- Восстановить `MAP_DATA` из `MAP_BACKUP`
- Установить `MOVE_ENABLE = 1`
- Обновить дисплей
- `TIMER_RUN` не изменять (таймер продолжает идти)

## События таймера и рекордов

### При EV_TIMER_TICK (прошла одна секунда)
- Если `TIMER_RUN == 1` и `GAME_STATE == 0`:
  - `TIMER_SEC = TIMER_SEC + 1`
  - Если `TIMER_SEC == 60`:
    - `TIMER_SEC = 0`
    - `TIMER_MIN = TIMER_MIN + 1`
  - Обновить дисплей (отображение времени)
  - Если `TIME_LIMIT_MODE == 1` и `TIMER_MIN >= 5` → вызвать `EV_TIME_LIMIT_REACHED`

### При EV_TIME_LIMIT_REACHED (достигнут лимит времени)
- Установить `GAME_STATE = 2` (поражение)
- Установить `TIMER_RUN = 0`
- Вызвать `EV_GAME_OVER`

### При EV_GAME_WIN (победа в игре)
- Остановить все движения (мобы не двигаются)
- Заблокировать обработку кнопок движения
- Зажечь победный индикатор
- Если `TIMER_MIN` и `TIMER_SEC` лучше `BEST_TIME_MIN` и `BEST_TIME_SEC`:
  - Вызвать `EV_NEW_RECORD`
- Отобразить экран WIN с итоговым временем

### При EV_NEW_RECORD (установлен новый рекорд)
- `BEST_TIME_MIN = TIMER_MIN`
- `BEST_TIME_SEC = TIMER_SEC`
- Зажечь индикатор нового рекорда
- Обновить дисплей (отображение рекорда)

### При EV_GAME_OVER (поражение)
- Заблокировать обработку кнопок движения
- Зажечь индикатор поражения
- Отобразить экран GAME OVER
- Игрок остаётся на месте, мобы не двигаются

## События тумана и видимости

### При EV_FOG_UPDATE (обновление тумана)
- Если `FOG_ENABLE == 1`:
  - Определить `VIEW_RADIUS`: базовый (например, 2) или расширенный если `BONUS_ACTIVE == 1`
  - Обновить зону видимости на дисплее (показать только клетки в радиусе от игрока)

### При EV_RADIUS_EXPAND (расширение радиуса)
- Установить расширенный радиус видимости
- Вызвать `EV_FOG_UPDATE`

### При EV_RADIUS_NORMALIZE (восстановление радиуса)
- Установить базовый радиус видимости
- Вызвать `EV_FOG_UPDATE`

## Системные события

### При EV_CLOCK_TICK (тактовый сигнал)
- Обновить счётчики:
  - Если `FREEZE_COUNTER > 0`:
    - `FREEZE_COUNTER = FREEZE_COUNTER - 1`
    - Если `FREEZE_COUNTER == 0` → вызвать `EV_FREEZE_END`
  - Если `BONUS_COUNTER > 0`:
    - `BONUS_COUNTER = BONUS_COUNTER - 1`
    - Если `BONUS_COUNTER == 0` и `BONUS_ACTIVE == 1` → вызвать `EV_BONUS_EXPIRED`
  - Если `TIMER_RUN == 1` и счётчик тактов достиг значения для 1 секунды → вызвать `EV_TIMER_TICK`
  - Если счётчик движения мобов достиг порога → вызвать `EV_MOB_MOVE`

### При EV_BLINK_UPDATE (обновление мигания)
- Инвертировать состояние мигания для клеток типа: ключ, ловушка, старт
- Обновить дисплей (мигающие элементы)

### При EV_MAP_RESTORE (восстановление карты)
- Скопировать содержимое `MAP_BACKUP` в `MAP_DATA`
- Обновить дисплей

### При EV_DISPLAY_UPDATE (обновление дисплея)
- Обновить VGA-выход (отображение лабиринта, игрока, мобов, тумана)
- Обновить 7-сегментные индикаторы (счётчик ключей, таймер, рекорд)
- Обновить светодиодные индикаторы (ошибка, заморозка, победа, поражение, рекорд)
